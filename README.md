# موقع Rov Wood 🪵

موقع لمصنع تصنيع كونتر الخشب — تصميم خشبي، صفحة أدمن محمية بكلمة سر، وقاعدة بيانات لإدارة الأقسام وأكواد التصميم.

---

## 1) إعداد قاعدة البيانات (Supabase) — مجاني

### أ. إنشاء المشروع
1. روح على https://supabase.com وسجّل حساب مجاني.
2. اضغط **New Project**.
3. اختار اسم للمشروع (مثلاً `rovwood`) وكلمة سر لقاعدة البيانات (احفظها في مكان آمن).
4. اختار المنطقة الأقرب لك، واضغط **Create new project** وانتظر دقيقتين لحد ما يجهز.

### ب. تشغيل أوامر SQL
1. من القائمة الجانبية، افتح **SQL Editor**.
2. اضغط **New query**.
3. افتح ملف `supabase-schema.sql` الموجود في هذا المشروع، انسخ محتواه بالكامل، والصقه في المحرر.
4. اضغط **Run**. هيتم إنشاء جدولين: `sections` و `design_codes`.

### ج. إعداد تخزين الصور (Storage)
1. من القائمة الجانبية، افتح **Storage**.
2. اضغط **Create a new bucket**.
3. اسم الـ bucket لازم يكون بالظبط: `design-images`
4. فعّل خيار **Public bucket** (مهم جدًا عشان الصور تظهر لأي زائر).
5. احفظ.
6. ارجع لـ **SQL Editor** وشغّل الجزء الخاص بـ Storage الموجود آخر ملف `supabase-schema.sql` (لو مكنش اتشغل من الأول).

### د. الحصول على مفاتيح الاتصال
1. من القائمة الجانبية: **Project Settings** (⚙️) > **API**.
2. هتلاقي:
   - **Project URL** → دا اللي هتحطه في `VITE_SUPABASE_URL`
   - **anon public key** → دا اللي هتحطه في `VITE_SUPABASE_ANON_KEY`

## 2) تشغيل المشروع محليًا (اختياري، للتجربة على جهازك)

```bash
# ثبّت الحزم
npm install

# انسخ ملف الإعدادات
cp .env.example .env
```

افتح ملف `.env` وحط فيه القيم اللي جبتها من Supabase:

```
VITE_SUPABASE_URL=https://xxxxxxxxxxxx.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-public-key-here
```

بعدين شغّل:

```bash
npm run dev
```

هيفتح الموقع على `http://localhost:5173`

---

## 3) رفع الموقع على Netlify

### الطريقة الأسهل (من غير GitHub)
1. شغّل الأمر ده عشان يجهز نسخة الإنتاج:
   ```bash
   npm run build
   ```
   هيتكوّن فولدر اسمه `dist`.
2. روح على https://app.netlify.com وسجّل دخول.
3. من الصفحة الرئيسية، اسحب فولدر `dist` وحطه في المكان اللي مكتوب عليه "Drag and drop your site output folder here".
4. الموقع هيترفع فورًا ويديك رابط.

⚠️ **مهم**: الطريقة دي مش بتاخد متغيرات `.env` تلقائيًا. لازم تضيفهم يدويًا:
1. من لوحة تحكم الموقع في Netlify: **Site configuration** > **Environment variables**.
2. اضغط **Add a variable** وضيف:
   - `VITE_SUPABASE_URL` = رابط مشروعك في Supabase
   - `VITE_SUPABASE_ANON_KEY` = المفتاح العام
3. بعد ما تضيفهم، لازم تعمل **Deploy** تاني عشان يقرأهم (من تبويب **Deploys** اضغط **Trigger deploy > Deploy site**).

### الطريقة الموصى بها (عبر GitHub — تحديثات تلقائية)
1. ارفع المشروع على GitHub (repo جديد).
2. من Netlify: **Add new site > Import an existing project**.
3. اختار GitHub ووصّل الـ repo بتاعك.
4. Build command: `npm run build`
   Publish directory: `dist`
   (دول متظبطين تلقائيًا من ملف `netlify.toml` الموجود في المشروع)
5. قبل ما تعمل Deploy، ضيف Environment Variables زي فوق بالظبط.
6. اضغط **Deploy site**.

من دلوقتي، أي تعديل تعمله وترفعه على GitHub هيتحدث على الموقع تلقائيًا.

## 4) استخدام لوحة التحكم (الأدمن)

1. روح على `https://your-site.netlify.app/admin`
2. كلمة السر الافتراضية: **`RovWood2019#`**

   ⚠️ **غيّرها قبل النشر**: افتح ملف `src/pages/AdminLogin.jsx` وغيّر قيمة `ADMIN_PASSWORD` في أول الملف بكلمة سر خاصة بيك.

3. بعد الدخول هتقدر:
   - **إضافة قسم جديد** (زي: سادة، فورمايكا، ميلامين، هاي جلوسي) من القائمة على اليمين.
   - اختيار أي قسم لعرض أكواده.
   - **إضافة كود تصميم جديد**: اكتب رقم الكود، وارفع صورة مصغرة (Thumbnail) وصورة حية بجودة أعلى للمعاينة الكبيرة.
   - حذف أي قسم أو كود.

الأكواد اللي بتضيفها هتظهر فورًا للزوار في الموقع، جوه صفحة القسم بتاعها، على شكل شبكة صور. أي زائر يضغط على "معاينة" هتفتح له الصورة الحية بشكل مكبّر (Lightbox).

## هيكل المشروع

```
rovwood/
├── src/
│   ├── components/     Header, Footer, Lightbox
│   ├── pages/           AdminLogin, AdminDashboard, SectionPage
│   ├── lib/              اتصال Supabase
│   ├── styles/          التصميم العام (الألوان والخطوط)
│   ├── App.jsx           الصفحة الرئيسية
│   └── main.jsx          نقطة الدخول + التوجيه (Routing)
├── supabase-schema.sql   أوامر إنشاء قاعدة البيانات
├── netlify.toml          إعدادات النشر على Netlify
└── .env.example          مثال لمتغيرات البيئة
```

## ملاحظات أمان

- كلمة سر الأدمن حاليًا مكتوبة داخل الكود (زي ما طلبت). ده كافي لموقع بسيط، لكن اعرف إن أي حد يقدر يشوف الكود المصدري (من DevTools) هيقدر يشوفها. لو عاوز حماية أقوى مستقبلًا، ممكن ننقل الباسورد لتحقق عبر السيرفر (Netlify Functions).
- قاعدة البيانات مفتوحة الكتابة لأي حد معاه الـ anon key (لأن التحقق بيتم من الواجهة بكلمة السر). ده قرار مقصود عشان يسهّل عليك الإدارة، لكن يعني إن مفيش طبقة حماية إضافية على مستوى الـ API نفسه.
